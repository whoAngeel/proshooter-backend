# Historias de Usuario - Sistema Pro Shooter

## Épica 1: Gestión de Usuarios y Autenticación


### HU-001: Registro de Usuario como Tirador
**Como** persona interesada en usar el sistema
**Quiero** poder registrarme proporcionando mis datos personales y de usuario
**Para** acceder a la plataforma y comenzar a utilizar las funcionalidades básicas como tirador

**Criterios de Aceptación:**
- El registro está abierto a cualquier persona sin intervención de un administrador
- El sistema solicita y valida los datos obligatorios (nombre, email, contraseña, etc.)
- Al registrarse, el usuario se crea automáticamente con el rol de Tirador
- Se inicializan los datos y estadísticas básicas del tirador
- El sistema verifica que el email/usuario sea único
- Se envía confirmación del registro exitoso al usuario

**Prioridad:** Must Have
**Estimación:** 8 Story Points
**Requerimiento Origen:** RF-01

---

### HU-002: Gestión de Información Personal
**Como** usuario del sistema
**Quiero** almacenar y actualizar mi información personal (nombre, dirección, teléfono)
**Para** mantener mis datos actualizados en el sistema

**Criterios de Aceptación:**
- El formulario debe permitir edición de todos los campos personales
- Debe validar formato de teléfono y dirección
- Los cambios deben guardarse exitosamente
- Debe mostrar confirmación de actualización

**Prioridad:** Must Have
**Estimación:** 5 Story Points
**Requerimiento Origen:** RF-02

---

### HU-003: Gestión de Datos Biométricos y Médicos
**Como** tirador registrado
**Quiero** registrar mis datos biométricos y médicos (peso, altura, tipo de sangre, alergias)
**Para** que el sistema considere estos factores en las evaluaciones y recomendaciones

**Criterios de Aceptación:**
- El formulario debe incluir campos para peso, altura, tipo de sangre y alergias
- Debe validar rangos apropiados para datos biométricos
- La información médica debe ser opcional pero recomendada
- Debe permitir actualización posterior de estos datos

**Prioridad:** Should Have
**Estimación:** 5 Story Points
**Requerimiento Origen:** RF-03

---

### HU-004: Promoción de Usuarios
**Como** administrador del sistema
**Quiero** promover usuarios entre roles (Tirador a Instructor, Instructor a Jefe de Instructor)
**Para** gestionar el crecimiento profesional según las reglas de negocio establecidas

**Criterios de Aceptación:**
- Solo se puede promover siguiendo la jerarquía establecida
- Debe validar requisitos previos para cada promoción
- El cambio de rol debe reflejarse inmediatamente en el sistema
- Debe mantener historial de promociones

**Prioridad:** Should Have
**Estimación:** 8 Story Points
**Requerimiento Origen:** RF-04

---

**Criterios de Aceptación:**

#### HU-15: Análisis Automático de Impactos y Puntuación
**Como:** Tirador
**Quiero:** Que el sistema detecte automáticamente los impactos en los blancos y calcule mi puntuación y métricas de agrupación respecto al centro
**Para:** Obtener una evaluación objetiva y visual de mi desempeño en cada sesión

**Criterios de Aceptación:**
1. El sistema procesa imágenes de los blancos automáticamente.
2. Detecta la posición de cada impacto en la imagen.
3. Calcula la puntuación de cada disparo en función de la distancia al centro del blanco.
4. Calcula métricas de agrupación y dispersión de los impactos.
5. Genera visualizaciones comprensibles de los resultados.

**Prioridad:** Must Have
**Estimación:** 21 Story Points
**Requerimiento Origen:** RF-14

---

#### HU-16: Visualización de Impactos Detectados en Ejercicio
**Como:** Tirador o instructor
**Quiero:** Ver los impactos detectados en la imagen de un ejercicio después del análisis automático
**Para:** Validar visualmente los resultados del análisis y comprender el desempeño en ese ejercicio

**Criterios de Aceptación:**
1. El usuario puede consultar los impactos detectados de un ejercicio mediante un endpoint pasando el ID del ejercicio.
2. El ejercicio debe tener una foto asociada y la foto debe tener un análisis realizado previamente.
3. Se muestran las posiciones de los impactos detectados sobre la imagen original.
4. Se visualizan los datos relevantes de cada impacto (posición, confianza, tipo).

**Prioridad:** Must Have
**Estimación:** 8 Story Points
**Requerimiento Origen:** RF-14

---

#### HU-17: Reanálisis Forzado de Impactos con Umbral de Confianza
**Como:** Tirador o instructor
**Quiero:** Forzar el reanálisis de la imagen de un ejercicio especificando un umbral de confianza
**Para:** Obtener un nuevo análisis que considere solo los impactos con una confianza mínima ajustable, en caso de que el análisis automático inicial no sea satisfactorio

**Criterios de Aceptación:**
1. El usuario puede solicitar el reanálisis de la imagen de un ejercicio mediante un endpoint, pasando el ID del ejercicio y el nuevo umbral de confianza.
2. Solo se consideran impactos cuya confianza sea igual o superior al umbral especificado.
3. El nuevo análisis reemplaza o complementa el análisis anterior y queda registrado.
4. Se notifica al usuario si el reanálisis fue exitoso y se muestran los nuevos resultados.

**Prioridad:** Should Have
**Estimación:** 8 Story Points
**Requerimiento Origen:** RF-14

---

#### HU-18: Actualización Automática de Estadísticas tras Evaluación
