
# 5.3 Implementación por Sprints

## 5.3.2 Sprint 1: Gestión de Usuarios Básica (Semanas 2-3)

### 5.3.2.1 Sprint Planning
En esta fase se definieron los objetivos y entregables del sprint, priorizando la implementación de la gestión básica de usuarios. Se establecieron los criterios de aceptación para el registro, autenticación y consulta de usuarios, así como la integración inicial de roles y permisos.

### 5.3.2.2 Desarrollo Realizado
Durante el sprint se desarrollaron los endpoints REST para el registro de usuarios, inicio de sesión (autenticación JWT), consulta de perfil y actualización de datos básicos. Se implementó la validación de datos con Pydantic y la gestión de contraseñas seguras mediante hashing. Además, se integró la lógica de roles de usuario y control de acceso en los endpoints principales.

### 5.3.2.3 Daily Scrums y Seguimiento
Se realizaron reuniones diarias (Daily Scrum) para monitorear el avance, identificar bloqueos y ajustar tareas. El seguimiento se documentó en la herramienta de gestión de proyectos, permitiendo la trazabilidad de los issues y el cumplimiento de los objetivos del sprint.

### 5.3.2.4 Sprint Review
Al cierre del sprint, se presentó una demo funcional de la gestión de usuarios ante el equipo y stakeholders. Se validó el cumplimiento de los criterios de aceptación y se recopilaron observaciones para mejoras futuras.

### 5.3.2.5 Sprint Retrospective
El equipo reflexionó sobre los aciertos y áreas de mejora del sprint. Se identificaron oportunidades para optimizar la comunicación, la estimación de tareas y la integración continua.

### 5.3.2.6 Burndown Chart y Métricas
Se generó un burndown chart para visualizar el progreso y la velocidad del equipo. Las métricas de cumplimiento y calidad se documentaron para su análisis en la retrospectiva.

### 5.3.2.7 Evidencias del Sprint
Se adjuntaron capturas de pantalla de los endpoints implementados, logs de pruebas exitosas y fragmentos de código relevantes como evidencia del trabajo realizado durante el sprint.
